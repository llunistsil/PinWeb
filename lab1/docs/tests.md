# Тестирование API

## Структура тестов

Тесты покрывают все основные эндпоинты приложения.

### Фикстуры (`conftest.py`)

- Подключение к тестовой БД
- Создание/удаление тестовой схемы
- HTTP клиент для запросов

### Покрытые модули

| Файл | Модуль |
|------|--------|
| `test_auth.py` | Аутентификация |
| `test_user.py` | Пользователи |
| `test_task.py` | Задачи |
| `test_priority.py` | Приоритеты |
| `test_schedule.py` | Расписание |
| `test_schedule_task.py` | Связь расписания и задач |
| `test_time_entry.py` | Учёт времени |
| `test_notification.py` | Уведомления |

## Запуск тестов

```bash
pytest tests/ -v
```

## Покрытие

Каждый модуль тестирует CRUD операции:
- Создание объекта
- Получение списка
- Получение по ID
- Обновление
- Удаление
